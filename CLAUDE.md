# CLAUDE.md

このファイルはClaudeがこのリポジトリで作業する際のガイドラインです。

---

## 特筆ルール：記事の文体・構成規則

以下のルールは `articles/` 配下の参考資料（官公庁向けコンサルティングレポート）を分析して抽出した、この記事の文体・構成の規則です。記事を作成・編集する際は必ずこのルールに従ってください。

---

### 1. ドキュメントの種類と対象読者

- **形式**: 官公庁・企業向けの調査・報告レポート（スライド形式のPDF）
- **対象**: 政策担当者、業界有識者、経営層など専門知識を持つ読者
- **目的**: 調査結果・分析内容・提言を論理的・簡潔に伝える

---

### 2. ページ・スライドの構成

各ページ（スライド）は以下の順序で構成する。

```
[ヘッダー]  章番号・章タイトル / 節番号・節タイトル
            例: 3. ＺＥＢ委員会の準備・運営 / 3.2 調査・分析｜ＺＥＢの普及状況

[サマリー文]  ページの要点を1〜2文で示す（◼ で始める）
              例: ◼ 〜を踏まえ、〜を整理した。

[本文]       箇条書き・表・図のいずれか、または組み合わせ

[フッター]   ページ番号（右下）、著作権表記・会社名（右下）
```

---

### 3. 文体・語調

#### 基本ルール
- **文末**: 「〜した。」「〜である。」「〜のとおりである。」「〜が求められる。」「〜することが重要である。」に統一する
- **語調**: である調（〜である、〜した）を基本とし、格調ある書き言葉を使う
- **一文の長さ**: 60〜80字程度を目安にする。長くなる場合は読点で区切るか、文を分割する
- **主語**: 「本事業では」「本調査において」など、文書・事業を主語にする。「私は」「我々は」は使わない
- **客観性**: 主観的・感情的な表現は一切使わない。事実・データ・論拠に基づいて記述する

#### 多用する表現パターン
| 用途 | 使用する表現 |
|---|---|
| 背景・目的を示す | 〜を踏まえ、〜を背景として、〜に向けて |
| 調査・実施を示す | 〜を実施した、〜を整理した、〜を検討した |
| 付加・補足を示す | 〜するとともに、〜に加え、なお〜 |
| 因果を示す | このため〜、これにより〜、〜ことから |
| 重要度を示す | 特に〜、〜が重要である、〜が求められる |
| 列挙の締め | 〜等、〜など（必ず最後に付ける） |
| 追加事項 | また〜、加えて〜 |

---

### 4. 見出し・ナンバリング

#### 章・節の番号体系
```
1.  大章（章タイトルのみのセクション区切りページを設ける）
1.1 節
1.1.1 項
```

#### 見出しの書き方
- 見出しは**体言止め**にする（例:「調査の背景・目的」「実施概要」「まとめ」）
- 動詞で終わらせない（「〜について説明する」ではなく「〜の概要」）
- 長い見出しには「・」または「〜に向けた〜」の形式を使う

---

### 5. 箇条書きの記号と階層

```
◼  第1階層（大項目、各ページの要点・サマリー文）
•  第2階層（中項目、説明・詳細）
➢  第3階層（小項目、具体例・補足）
⚫  ハイライト・結論のまとめ（囲み枠的な強調）
```

- 箇条書き1項目は1〜3行以内に収める
- 体言止め（名詞・動詞の連用形止め）を基本とする
- 各階層をむやみに深くしない（3階層まで）

---

### 6. 表の作成方法

#### 基本構造
- **必ずヘッダー行を設ける**（太字または網掛け）
- セルの内容は簡潔に（箇条書き・体言止め）
- 空欄には「ー」を使用する（N/A・空白・「なし」は使わない）
- 出典がある場合は表の下に「出所）〜より作成」と記載する

#### よく使う表の型
| 型 | 列構成の例 |
|---|---|
| プロセス整理 | 調査ステップ ｜ 目的 ｜ 作業内容 |
| スキルマップ | 項目 ｜ ポイント ｜ 知識 ｜ スキル |
| 比較表 | 手法 ｜ 特徴 ｜ メリット・デメリット |
| 開催実績 | 回数 ｜ 実施日時 ｜ 議事 |
| 調査対象 | No ｜ 企業 ｜ 業態 ｜ 業種 ｜ 実施日時 |

---

### 7. 図・ダイアグラムの扱い

- 図の直前に ◼ サマリー文を置き、図が何を示すかを先に説明する
- 図・チャートの下には「出所）〜より〇〇作成」を付記する
- フロー図・構造図は、矢印で関係性・プロセスを示す
- 複雑な情報は箇条書きより図（フロー・マトリクス）で表現する

---

### 8. 数字・記号の表記ルール

- **年度表記**: 「令和○年度」（和暦）を基本とし、西暦を括弧で添えることもある（例: 令和6年度（2024年度））
- **数字**: 半角数字を基本とする（全角は固有名詞・法令名のみ）
- **パーセント**: 「○○%」（半角・記号）
- **順位・番号**: 「第○回」「①②③」「1. 2. 3.」を使い分ける
- **省略**: 初出は正式名称+略称を示す（例: 「事業承継・引継ぎ支援センター（以下「センター」という。）」）

---

### 9. サマリー文の書き方

各ページ冒頭の1〜2行で「このページで何を伝えるか」を先に示す。以下のパターンを参考にする。

```
◼ 〜について、〜を踏まえ、〜を整理した。
◼ 〜の調査の結果、〜であることが確認された。
◼ 〜に向けて、〜を実施するとともに、〜を行った。
◼ 本事業は、〜を目的として、〜を実施するものである。
◼ 以下には〜の概要を示す。
```

- 結論先行（BLUF: Bottom Line Up Front）で書く
- 詳細は本文（箇条書き・表）で補足する

---

### 10. 避けるべき表現

以下の表現は使用しない。

| 避けるべき表現 | 理由 | 代替表現 |
|---|---|---|
| 〜と思います | 主観的・不確か | 〜と考えられる、〜が想定される |
| 〜でしょう、〜かもしれません | 推量・曖昧 | 〜する可能性がある、〜と見込まれる |
| 〜させていただきます（多用） | 過度な謙譲 | 〜する、〜を実施する |
| いい感じ、うまくいく | 曖昧な評価表現 | 適切な、効果的な、〜を達成する |
| ちょっと、少し | 程度が不明確 | 一定程度、軽微な、〇〇%程度 |
| 〜ですが、〜ですので | 話し言葉調 | 〜であるが、〜であるため |
| 〜してください（本文中） | 指示口調 | 〜することが求められる、〜する必要がある |
| 非常に、とても（感嘆的） | 感情的・主観的 | 相当程度、大幅に、顕著に |
| 〜でしょうか？（本文中） | 問いかけ口調 | 記述・断言形式に変換する |

---

### 11. 全体構成のテンプレート

```
表紙
  └ 発注者名
  └ 事業名（正式名称）
  └ 報告書タイトル
  └ 作成者（会社名・部署名）
  └ 年月

目次
  └ 章・節の一覧とページ番号

本文
  └ 1. 事業の全体像（背景・目的・事業内容）
  └ 2. 〔調査テーマA〕の準備・運営
      └ 2.1 実施概要
      └ 2.2 調査・分析
          └ 2.2.1 〔小テーマ〕
          └ 2.2.2 〔小テーマ〕
          └ 2.2.3 まとめ
      └ 2.3 取りまとめ資料
  └ 3. 〔調査テーマB〕の準備・運営
      └ （同様の構成）

付属資料
```

---

### 12. まとめ・結論のページ

- 各章末尾に「まとめ」のページを設ける
- 本文で扱った論点を箇条書きで再整理する
- 「今後の課題」「次のステップ」を示す場合は別ブロックで記載する
- 「〜が重要である。」「〜に向けた取組が求められる。」で締めくくる
