# プロンプトの基礎

## はじめに

AIツールを使い始めたとき、「なんとなく話しかけてみたけど、思った通りの返答が来なかった」という経験はありませんか。

実は、AIへの伝え方には少しコツがあります。そのコツを知っておくだけで、AIとのやりとりがぐっと楽になります。この記事では、そのコツの基本となる「プロンプト」について、基礎からやさしく説明していきます。

## プロンプトとは何か

プロンプト（Prompt）とは、AIに入力する文章のことです。日本語では「指示文」や「質問文」と呼ばれることもあります。

ChatGPTやClaude、Geminiなどのチャット型AIでいえば、あなたがテキストボックスに打ち込む言葉がそのままプロンプトにあたります。つまり、AIへの「話しかけ方」そのものです。

ここで一つ知っておきたいのは、**同じAIでも、プロンプトの書き方によって返ってくる内容が大きく変わる**ということです。

```
あまり良くない例: 「記事を書いて」

より良い例: 「初心者向けに、Pythonのリスト操作について300字程度で解説する記事を書いてください」
```

前者では何について、誰向けに、どのくらいの量を書けばよいかがAIに伝わりません。後者のように具体的に伝えると、期待に近い回答が返ってきやすくなります。

## プロンプトの基本構成要素

効果的なプロンプトには、いくつかの要素が含まれていることが多いです。すべてを毎回入れる必要はありませんが、迷ったときのチェックリストとして活用してみてください。

### 1. 役割（Role）

AIに「どんな立場で答えてほしいか」を伝える部分です。役割を与えると、それに沿ったトーンや視点で回答してもらえます。

```
あなたは経験豊富なソフトウェアエンジニアです。
```

### 2. コンテキスト（Context）

背景情報や前提条件を伝える部分です。「今どういう状況なのか」をAIに知らせることで、的外れな回答を避けられます。

```
私はWebアプリケーションを開発しており、ユーザー認証機能を実装しようとしています。
```

### 3. タスク（Task）

「具体的に何をしてほしいか」を伝える部分です。プロンプトの中でもっとも重要な要素といえます。

```
JWTを使ったログイン機能のサンプルコードを書いてください。
```

### 4. 形式（Format）

出力の形式や長さを指定する部分です。「箇条書きで」「表にして」「300字以内で」といった指定がここに当たります。

```
コードブロックを使い、各行にコメントを付けてください。
```

### 5. 制約（Constraints）

「これはやらないでほしい」「この条件は守ってほしい」といった縛りを伝える部分です。

```
外部ライブラリは使わず、標準ライブラリのみで実装してください。
```

## 良いプロンプトの特徴

### 明確であること

曖昧な表現はなるべく避けて、具体的に書くと伝わりやすくなります。

| 曖昧な表現 | 具体的な表現 |
|---|---|
| 「ちょっと説明して」 | 「200字以内で要点を3つ挙げて説明してください」 |
| 「良い感じにして」 | 「ビジネス向けに丁寧な敬語で書き直してください」 |
| 「コードを直して」 | 「この関数のパフォーマンスを改善し、処理速度を上げてください」 |

### 具体的であること

期待する出力のイメージを、できるだけ具体的に伝えましょう。「こんな感じで出してほしい」という例を見せるのも効果的です。

### 適切な長さであること

プロンプトは長ければ良いというわけではありません。必要な情報をしっかり盛り込みつつ、余計な言葉は省く意識が大切です。長すぎると、重要な指示がAIに伝わりにくくなることもあります。

### 繰り返し使えること

何度使っても安定した出力が得られるプロンプトは、それだけ完成度が高いといえます。同じプロンプトを試しながら少しずつ改善していくのが上達の近道です。

## 主なプロンプト技法

少し慣れてきたら、以下のような技法も取り入れてみましょう。

### Zero-Shot プロンプティング

例を示さずに、直接タスクを指示する方法です。シンプルなタスクに向いています。

```
次の文章を英語に翻訳してください。

文章: 今日は良い天気ですね。
```

### Few-Shot プロンプティング

いくつかの例を見せてから、タスクを指示する方法です。「こういうパターンで答えてほしい」という形を示せるので、出力の精度が上がりやすくなります。

```
以下の形式で感情を分類してください。

入力: 「映画がとても面白かった！」
出力: ポジティブ

入力: 「電車が遅れてイライラした」
出力: ネガティブ

入力: 「明日は会議がある」
出力: 中立

入力: 「新しいプロジェクトが始まってワクワクしている」
出力:
```

### Chain-of-Thought（思考の連鎖）

「ステップバイステップで考えてください」と指示することで、AIが順を追って考えるようになります。計算や論理的な問題で特に効果的です。

```
以下の問題をステップバイステップで解いてください。

問題: 太郎さんは1個100円のリンゴを5個と、1個150円のオレンジを3個買いました。
1000円払った場合、おつりはいくらですか？

ステップ1: リンゴの合計金額を計算する
ステップ2: オレンジの合計金額を計算する
ステップ3: 合計金額を計算する
ステップ4: おつりを計算する
```

### システムプロンプト

チャットシステムでは、会話の最初にAIの基本的な振る舞いを設定できます。これを「システムプロンプト」と呼びます。カスタマーサポートbotや特定の用途に特化したAIを作るときによく使われます。

```
あなたは親切で丁寧なカスタマーサポート担当者です。
常に敬語を使い、ユーザーの問題解決を最優先に考えてください。
個人情報には触れず、わからないことは「確認します」と伝えてください。
```

## よくある失敗パターン

プロンプトを書き始めた頃によく陥りがちな失敗を紹介します。心当たりがあっても、少し直すだけで改善することが多いので安心してください。

### 指示が曖昧すぎる

```
NG: 「いい感じに書いて」
OK: 「読者が初心者であることを想定し、専門用語を避けて平易な言葉で説明してください」
```

「いい感じ」はAIには伝わりません。どんな状態が「いい」のかを具体的に言葉にしてみましょう。

### 複数のタスクを一度に詰め込みすぎる

一つのプロンプトにたくさんの要求を詰め込むと、どれも中途半端な結果になりやすいです。複雑な作業は、いくつかのステップに分けて順番に指示するほうがうまくいきます。

### 否定形で指示する

「〜しないで」という指示より「〜してください」という指示のほうが、AIには伝わりやすい傾向があります。

```
NG: 「難しい言葉を使わないで」
OK: 「中学生でも理解できる、やさしい言葉で説明してください」
```

### 出力の形式を指定しない

形式を指定しないと、毎回異なる構成で返ってくることがあります。使いやすい形が決まっているなら、最初から指定しておきましょう。

```
NG: 「メリットとデメリットを教えて」
OK: 「メリットとデメリットをそれぞれ箇条書きで3点ずつ挙げてください」
```

## 実践例

ここでは、実際に使えるプロンプトの例をいくつか紹介します。そのままコピーして、自分のケースに合わせて書き換えてみてください。

### 例1: コードレビューの依頼

```
あなたはシニアPythonエンジニアです。
以下のコードをレビューし、問題点と改善案を指摘してください。

評価の観点:
- 可読性
- パフォーマンス
- セキュリティ
- Pythonのベストプラクティス

```python
def get_user(id):
    db = connect_db()
    result = db.query("SELECT * FROM users WHERE id=" + id)
    return result
```
```

### 例2: 文章の要約

```
以下の文章を読んで、下記の形式で要約してください。

【要約形式】
- 要点（3行以内）
- キーワード（5つ）
- 対象読者

文章:
（要約したい文章をここに貼り付けてください）
```

### 例3: アイデア出し

```
ECサイトのユーザー離脱率を下げるための施策を考えてください。

条件:
- 予算は限られている（無料または低コストの施策を優先）
- 技術的な実装を伴うものと伴わないものの両方を含める
- 実施の優先度（高・中・低）も示すこと

10個のアイデアをリスト形式で出力してください。
```

## まとめ

この記事で紹介した、良いプロンプトを書くためのポイントを振り返りましょう。

1. **役割・コンテキスト・タスク・形式・制約**の5要素を意識して書く
2. **曖昧な表現を避け**、具体的な言葉で伝える
3. **例示（Few-Shot）**を使うと、出力のパターンを揃えやすい
4. **「ステップバイステップで」**と一言加えるだけで、複雑な問題の精度が上がることがある
5. **出力形式を指定する**と、毎回使いやすい形で返ってくる

プロンプトを書くのは、最初から完璧にできなくても大丈夫です。「なんか違うな」と思ったら少し言葉を変えてみる、その繰り返しで自然と上手くなっていきます。まずは気軽に試してみてください。
