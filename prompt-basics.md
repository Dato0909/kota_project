# プロンプトの基礎

## はじめに

プロンプトとは、AIに対して与える指示や質問のことです。適切なプロンプトを書くことで、AIから期待通りの回答や出力を得ることができます。本記事では、プロンプトの基本的な概念から実践的なテクニックまでを解説します。

## プロンプトとは何か

プロンプト（Prompt）は、大規模言語モデル（LLM）などのAIシステムに対して入力するテキストです。人間がAIと対話する際の「話しかけ方」に相当します。

プロンプトの質は、AIの出力品質に直接影響します。同じAIモデルでも、プロンプトの書き方によって出力結果が大きく変わります。

```
悪い例: 「記事を書いて」
良い例: 「初心者向けに、Pythonのリスト操作について300字程度で解説する記事を書いてください」
```

## プロンプトの基本構成要素

効果的なプロンプトは、以下の要素を含むことが多いです。

### 1. 役割（Role）

AIにどのような立場で回答してほしいかを指定します。

```
あなたは経験豊富なソフトウェアエンジニアです。
```

### 2. コンテキスト（Context）

背景情報や前提条件を提供します。

```
私はWebアプリケーションを開発しており、ユーザー認証機能を実装しようとしています。
```

### 3. タスク（Task）

具体的に何をしてほしいかを明確に伝えます。

```
JWTを使ったログイン機能のサンプルコードを書いてください。
```

### 4. 形式（Format）

出力の形式や長さを指定します。

```
コードブロックを使い、各行にコメントを付けてください。
```

### 5. 制約（Constraints）

守ってほしい条件や禁止事項を指定します。

```
外部ライブラリは使わず、標準ライブラリのみで実装してください。
```

## 良いプロンプトの特徴

### 明確性

曖昧な表現を避け、具体的な指示を書きます。

| 曖昧な表現 | 明確な表現 |
|---|---|
| 「ちょっと説明して」 | 「200字以内で要点を3つ挙げて説明してください」 |
| 「良い感じにして」 | 「ビジネス向けに丁寧な敬語で書き直してください」 |
| 「コードを直して」 | 「この関数のパフォーマンスを改善し、時間計算量をO(n²)からO(n)にしてください」 |

### 具体性

期待する出力のイメージを具体的に伝えます。例を示すのも効果的です。

### 適切な長さ

プロンプトは必要な情報を含む範囲で、できるだけ簡潔にします。長すぎると重要な指示が埋もれることがあります。

### 反復可能性

同じプロンプトを複数回使っても安定した出力が得られるように書くことが理想です。

## 主なプロンプト技法

### Zero-Shot プロンプティング

例を示さずに直接タスクを指示する手法です。

```
次の文章を英語に翻訳してください。

文章: 今日は良い天気ですね。
```

### Few-Shot プロンプティング

いくつかの例（例示）を示してからタスクを指示する手法です。AIが出力のパターンを学習します。

```
以下の形式で感情を分類してください。

入力: 「映画がとても面白かった！」
出力: ポジティブ

入力: 「電車が遅れてイライラした」
出力: ネガティブ

入力: 「明日は会議がある」
出力: 中立

入力: 「新しいプロジェクトが始まってワクワクしている」
出力:
```

### Chain-of-Thought（思考の連鎖）

AIに段階的に考えさせることで、複雑な問題の精度を上げる手法です。

```
以下の問題をステップバイステップで解いてください。

問題: 太郎さんは1個100円のリンゴを5個と、1個150円のオレンジを3個買いました。
1000円払った場合、おつりはいくらですか？

ステップ1: リンゴの合計金額を計算する
ステップ2: オレンジの合計金額を計算する
ステップ3: 合計金額を計算する
ステップ4: おつりを計算する
```

### システムプロンプト

AIの基本的な振る舞いや人格を設定するための指示です。チャットシステムでは会話の冒頭に配置されます。

```
あなたは親切で丁寧なカスタマーサポート担当者です。
常に敬語を使い、ユーザーの問題解決を最優先に考えてください。
個人情報には触れず、わからないことは「確認します」と伝えてください。
```

## よくある失敗パターン

### 指示が曖昧すぎる

```
NG: 「いい感じに書いて」
OK: 「読者が初心者であることを想定し、専門用語を避けて平易な言葉で説明してください」
```

### 複数のタスクを一度に詰め込みすぎる

一つのプロンプトに多くのタスクを詰め込むと、どれも中途半端になりやすいです。タスクを分けて段階的に指示しましょう。

### 否定形での指示

何をしてほしくないかより、何をしてほしいかを伝える方が効果的です。

```
NG: 「難しい言葉を使わないで」
OK: 「中学生でも理解できる簡単な言葉を使って説明してください」
```

### 出力形式を指定しない

形式を指定しないと、期待と異なる構造で出力されることがあります。

```
NG: 「メリットとデメリットを教えて」
OK: 「メリットとデメリットをそれぞれ箇条書きで3点ずつ挙げてください」
```

## 実践例

### 例1: コードレビューの依頼

```
あなたはシニアPythonエンジニアです。
以下のコードをレビューし、問題点と改善案を指摘してください。

評価の観点:
- 可読性
- パフォーマンス
- セキュリティ
- Pythonのベストプラクティス

```python
def get_user(id):
    db = connect_db()
    result = db.query("SELECT * FROM users WHERE id=" + id)
    return result
```
```

### 例2: 文章の要約

```
以下の文章を読んで、下記の形式で要約してください。

【要約形式】
- 要点（3行以内）
- キーワード（5つ）
- 対象読者

文章:
（要約したい文章をここに貼る）
```

### 例3: アイデア出し

```
ECサイトのユーザー離脱率を下げるための施策を考えてください。

条件:
- 予算は限られている（無料または低コストの施策を優先）
- 技術的な実装を伴うものと伴わないものの両方を含める
- 実施の優先度（高・中・低）も示すこと

10個のアイデアをリスト形式で出力してください。
```

## まとめ

効果的なプロンプトを書くポイントをまとめます。

1. **役割・コンテキスト・タスク・形式・制約**を意識して構成する
2. **具体的かつ明確**な言葉を使う
3. **例示**を活用してAIに期待するパターンを示す
4. **段階的な思考**を促すことで複雑なタスクの精度が上がる
5. **出力形式**を明示することで使いやすい回答が得られる

プロンプトの書き方はスキルであり、練習によって上達します。試行錯誤しながら、自分のユースケースに合った最適なプロンプトを見つけていきましょう。
